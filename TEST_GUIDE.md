## 快速测试步骤

### 方式1：浏览器测试（推荐）

1. **打开浏览器**
   ```
   http://localhost:6001
   ```

2. **上传测试PPT**
   - 点击"选择文件"

   - 选择讲稿时长（建议60分钟）

3. **观察进度**
   - 页面会显示"正在分析PPT结构..."（第一阶段）

   - 观察浏览器控制台（F12）查看实时日志

4. **查看结果**
   - 生成完成后，下载生成的PPT和TXT文件
   - 查看讲稿是否消除了重复的"同学们"
   - 检查术语翻译是否统一

## 常见问题排查

### Q1: 服务启动失败

**症状**: 端口被占用或无法启动

**解决**:
```powershell
# 查找占用端口的进程
Get-NetTCPConnection -LocalPort 6001 | Select-Object OwningProcess

# 停止进程
Stop-Process -Id <进程ID> -Force

# 重新启动
uv run python run.py
```

### Q2: 第一阶段超时

**症状**: 长时间停留在"正在分析PPT结构..."

**可能原因**:
- AI模型响应慢
- 网络连接问题
- PPT文件过大

**解决**:
- 查看日志确认卡在哪一步
- 尝试切换到更快的模型（如Ollama）
- 减小PPT页数

### Q3: JSON解析失败

**症状**: 日志显示"解析全局分析JSON失败"

**影响**: 系统会自动使用默认上下文，不会中断流程

**解决**:
- 这是正常的容错行为
- 查看 `app.log` 了解详细错误
- 尝试不同的AI模型

### Q4: 讲稿仍有重复开场白

**症状**: "同学们"仍然频繁出现

**可能原因**:
- AI未严格遵循prompt
- 章节首页确实需要开场

**建议**:
- 这是AI的自然行为，在关键节点（首页、章节开始）有开场是合理的
- 如果普通页面也有开场，可以调整prompt温度参数
