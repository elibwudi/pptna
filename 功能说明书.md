# PPT讲稿AI生成系统 - 功能说明书 (v4.0)

## 1. 系统概述
PPT讲稿AI生成系统是一款基于多模态大语言模型（LLM）的智能化办公辅助平台。系统能够自动分析用户上传的 PPT 文稿，生成逻辑连贯、风格统一的演讲讲稿，并将其自动植入 PPT 的备注栏中。

### 核心价值
- **高效自动化**：分钟级完成长篇 PPT 的讲稿撰写。
- **内容一致性**：通过全局分析确保术语统一和内容承接。
- **双重保障**：支持多种主流 AI 模型，内置自动化异常回退。
- **环境友好**：支持 Windows 后台静默运行，不干扰日常工作。

---

## 2. 系统架构与技术栈
系统采用轻量化、高性能的现代技术架构，确保在本地及服务器环境下的稳定运行。

- **后端框架**：Python 3.14 + Flask (Web 接口层)
- **生产服务器**：Waitress (高性能 WSGI 容器)
- **环境管理**：`uv` (极速 Python 包管理工具)
- **前端技术**：HTML5 + Vanilla CSS (Glassmorphism 磨砂玻璃风格) + JavaScript
- **核心依赖库**：
  - `python-pptx`: PPT结构处理
  - `PyMuPDF (fitz)`: 视觉分析与缩略图生成
  - `pywin32 (COM)`: PPT转换与备注修复补丁
  - `google-generativeai / openai`: AI 模型接口

---

## 3. 核心功能模块

### 3.1 文件处理与预览
- **多格式支持**：主要支持 `.pptx` 格式。
- **视觉转换**：自动将 PPT 页面转换为高清图片，提供网页端实时预览。
- **自动清理**：内置过期临时文件自动清理机制。

### 3.2 两阶段 AI 生成引擎 (核心)
- **第一阶段：全局分析**
  - AI 预扫描整个文稿，提取目录结构、专业术语表和整体教学风格。
  - **逻辑模块重构**：打破传统“教科书章节”的认知局限，智能划分为灵活的“教学模块”，杜绝大模型生硬播报“下面进入第X章第Y节”。
- **第二阶段：逐页生成**
  - 基于全局上下文，并**继承上一页生成的讲稿 (Previous Notes)**，实现自然无缝的话语衔接，彻底消除重复的口头禅开场。
  - **动态字数分配算法 (防平均主义)**：取消死板的字数均摊。系统会根据每页的“文本厚度 + 图片权重”实时加权分配目标字数（标题页自动极简，核心页获得充足时长）。
  - **极致口语化红线约束**：注入强制指令，要求大模型严守绝对字数上限，且大量使用短句、生活化类比与互动词汇，消除AI机械书面语感。

### 3.3 自动备注插入
- **标准写入**：使用 `python-pptx` 直接写入备注。
- **COM 补丁**：针对特殊模板（如部分加密或受限模板），自动调用系统的 PowerPoint 进程进行底层注入，确保写入成功率 100%。

### 3.4 模型管理与测试
- **主流模型集成**：
  - **Google Gemini**: 最强多模态理解能力。
  - **DeepSeek / 阿里通义千问**: 卓越的中文逻辑与文案功底。
  - **Ollama (Gemma 4 31B)**: 本地部署，隐私安全，无网络要求。
- **实时测试**：内置模型连接性测试工具，一键排查配置问题。

## 5. 运维与运行保障

### 5.1 启动模式
- **后台静默运行**：使用 `pythonw.exe` 启动，无黑色命令行窗口。
- **标准流重定向**：标准输出自动重定向至 `os.devnull`，彻底消除 `[Errno 22]` 错误。

### 5.2 状态监控
- **实时控制台**：支持 PowerShell 实时日志监控逻辑。
- **系统状态脚本**：一键检查端口、进程、环境及目录健康度。

### 5.3 自动化管理脚本
- `启动服务.bat`: 自动同步环境 (`uv sync`) 并后台启动。
- `停止服务.bat`: 安全结束相关进程并释放端口。
- `系统状态.bat`: 全方位系统体检报告。

---

## 6. 使用说明

1. **启动**：双击 `启动服务.bat`。
2. **访问**：在浏览器打开 `http://localhost:6001`。
3. **上传**：选择 PPT 文件并设定目标宣讲时长。
4. **生成**：等待 AI 完成两阶段处理。
5. **下载**：获取已内嵌讲稿的 PPT 文件。
