# 两阶段PPT讲稿生成系统 - 使用说明

## 概述

系统现已升级为**两阶段生成架构**，通过全局分析确保讲稿的一致性和连贯性。

### 核心改进 (V4.0 全新升级)

✅ **动态字数分配算法** - 告别平均主义，根据每页内容厚度动态加权计算目标字数，标题页等过渡页享受智能极简豁免权。
✅ **极致口语化与绝对红线** - 加入超强硬的口语化约束和绝对字数上限，消除大模型的书面语感和毫无节制的“话痨”凑字数。
✅ **逻辑模块重构** - 摒弃死板的“第X章”，按认知逻辑划分模块，实现无痕顺滑的话语过渡。
✅ **智能消除重复开场** - 将上一页的实际生成内容 (`previous_notes`) 注入下一页上下文，彻底告别每页都喊“同学们”。
✅ **术语翻译统一** - 专业术语在整个PPT中保持一致。

---

## 工作原理

### 第一阶段：全局分析

在生成具体讲稿之前，系统会：
1. 快速扫描所有幻灯片内容
2. 生成全局教学规划，包括：
   - **章节划分**：自动识别3-8个逻辑章节
   - **术语表**：提取10-30个专业术语并统一翻译
   - **风格指南**：确定教学语气、比喻风格、受众水平
   - **核心主题**：提炼2-4个核心主题

**耗时**：约1-2分钟（取决于PPT大小）

### 第二阶段：逐页生成

基于全局上下文与动态分配的目标字数，为每一页生成讲稿：
- **字数红线控制**：严格按动态分配字数执行，杜绝短页强行凑字。
- **无缝衔接**：继承上一页的话语 (`previous_notes`) 实现智能承上启下。
- **强口语化**：引入大量生活化类比、短句与互动词汇。
- **术语纪律**：严格遵守术语表，确保翻译一致。

**耗时**：与原版相同

## 配置选项

编辑 `server_config.json` 文件：

```json
{
    "llm_provider": "qwen",
    "generation_mode": "two_stage",  // "two_stage" 或 "legacy"
    "two_stage_config": {
        "enabled": true,               // 是否启用两阶段生成
        "min_slides_for_two_stage": 3,  // 最少页数才使用两阶段（默认3页）
        "max_slides_for_full_analysis": 50, // 完整分析的最大页数
        "max_chars_per_slide": 200,    // 每页摘要最大字符数
        "max_terms_in_glossary": 30    // 术语表最大术语数
    }
}
```

### 配置说明

**generation_mode**（生成模式）
- `two_stage`：使用两阶段生成（推荐）
- `legacy`：使用旧版单阶段生成（回退选项）

**min_slides_for_two_stage**
- 小于此页数的PPT跳过第一阶段，直接使用默认上下文
- 默认值：3页
- 建议：保持默认值，小型PPT无需全局分析

**max_slides_for_full_analysis**
- 超过此页数的PPT会进行智能采样（前5页+中间采样+最后5页）
- 默认值：50页
- 建议：大型PPT（50+页）可适当提高此值

## 使用示例

### 示例1：中型PPT（10-30页）

**输入**：20页的技术培训PPT

**第一阶段输出**：
- 识别出4个章节：概念介绍、技术原理、实践案例、总结
- 提取15个专业术语（如API、SDK、Deployment等）
- 风格定位：专业严谨，使用生活化类比

**第二阶段特点**：
- 第1页：引人入胜的开场
- 第6页（章节2开始）："我们开始新的章节：技术原理"
- 第10页（章节2结束）：总结技术原理核心要点
- 中间页面：自然过渡，不重复"同学们"

### 示例2：小型PPT（1-2页）

**行为**：自动跳过第一阶段，直接生成

**理由**：页面太少，无需全局分析

### 示例3：大型PPT（50+页）

**行为**：第一阶段智能采样，控制分析时间

**采样策略**：
- 完整分析前5页
- 中间每隔N页采样1页
- 完整分析最后5页
- 总输入控制在5000字以内

## 常见问题

### Q1: 如何回退到旧版？

**A**: 修改 `server_config.json`：
```json
{
    "generation_mode": "legacy"
}
```

### Q2: 第一阶段失败会怎样？

**A**: 系统会自动使用默认上下文继续生成，不会中断流程：
- 默认1个大章节
- 无术语表
- 默认风格：专业且亲切
- 记录错误日志

### Q3: 可以调整AI生成的风格吗？

**A**: 可以。编辑生成的 `global_analysis.json` 文件，调整：
- `style_guide.tone`：改变语气
- `style_guide.metaphor_style`：改变比喻风格
- `glossary.terms`：添加或修改术语

然后重新上传PPT，系统会使用修改后的配置（需要手动实现此功能）。

### Q4: 为什么有些页面还是重复开场？

**A**: 可能原因：
1. AI未严格遵循prompt - 可通过调整temperature参数改善
2. 章节识别不准确 - 可手动调整 `global_analysis.json` 中的章节划分
3. 首页或章节首页 - 这些页面确实需要"同学们"等开场

### Q5: 支持哪些AI模型？

**A**: 全部4个模型均支持两阶段生成：
- ✅ Gemini（多模态）
- ✅ DeepSeek（文本）
- ✅ Qwen（文本）
- ✅ Ollama（多模态）

**注意**：第一阶段只需要文本，多模态优势不明显。

## 技术细节

### Prompt改进

**旧版问题**：
```
在上一页，我们刚刚讲了: "{previous_notes}"
→ 导致常说"同学们，在上一页我们讲了..."
```

**新版方案**：
```
# 首页
"这是课程的第一页。请设计一个引人入胜的开场。"

# 章节开始
"我们开始新的章节：『{章节名}』。请介绍学习目标。"

# 章节结束
"这是章节的最后一页。请总结核心要点。"

# 普通页面
"请继续讲解新内容，保持连贯性。"
```

### 边缘情况处理

1. **小型PPT（<3页）**：跳过第一阶段
2. **大型PPT（>50页）**：智能采样
3. **第一阶段失败**：Fallback到默认上下文
4. **JSON解析失败**：容错处理，使用默认值
